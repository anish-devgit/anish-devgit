<details open>
<summary><b>🧠 Project 1 — Hybrid Search RAG Engine</b> &nbsp;|&nbsp; <code>Python · LangChain · FAISS · GPT-4 · Docker</code></summary>

<br/>

> A production-grade Retrieval-Augmented Generation system built to query 500+ pages of technical documentation with high accuracy — zero hallucinations.

| Metric | Result |
|--------|--------|
| Retrieval Accuracy | **+40%** over baseline keyword search |
| Search Method | Hybrid: FAISS (semantic) + BM25 (keyword) + RRF fusion |
| Context Quality | Semantic Chunking — no fixed-size splits |
| Deployment | Dockerized FastAPI inference API |
| LLM | OpenAI GPT-4 with citation-enforced prompts |

**Key Engineering Decisions:**
- 🔀 **Reciprocal Rank Fusion (RRF)** to merge dense + sparse results — not just a naive concatenation
- 🧩 **SemanticChunker** instead of fixed-size splits — preserves context boundaries
- 🐳 **Docker containerization** — single `docker compose up` to run
- 🚫 **Anti-hallucination prompt design** — model instructed to cite source + page, or say "I don't know"

<summary><b>📈 Project 3 — Algorithmic Trading Monitor</b> &nbsp;|&nbsp; <code>MERN Stack · WebSockets · Chart.js · MongoDB</code></summary>

<br/>

> A high-frequency financial analytics dashboard that streams live market tick data in real time and auto-calculates technical indicators to surface buy/sell signals.

| Metric | Result |
|--------|--------|
| Data Latency | **< 100ms** via WebSocket streaming |
| Indicators | RSI(14), SMA-20, EMA-50, Bollinger Bands |
| Price Simulation | Geometric Brownian Motion — realistic price movement |
| Stack | React · Node.js · Express · MongoDB · WebSockets |
| Signals | Auto-alerts when RSI > 70 (overbought) or RSI < 30 (oversold) |

**Key Engineering Decisions:**
- ⚡ **Raw `ws` library** over Socket.io — lower overhead, measurably better latency
- 🔢 **GBM price simulation** — mathematically grounded, not random noise
- 🖼️ **Chart.js `chart.update()`** — appends data points without full re-render (critical for 100ms refresh)
- 📡 **Symbol-level subscription model** — clients only receive data for tickers they're watching
